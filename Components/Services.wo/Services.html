Les services de l'application ServAut
(version )

 Direct Actions

Le URL pour appeler une "direct action" d'une application est le suivant :
  http://mon.serveur.com/cgi-bin/WebObjects/UneApplication.woa/wa/nomAction?parametre1=valeur1&param2=val2&...

 Action : Cryptage de dictionnaire de connexion

Description :

Cette action affiche la page avec le formulaire permettant de définir et de crypter un dictionnaire de connexion à une base de données. La valeur cryptée généré peut être mise dans le fichier de configuration de l'application (paramètre APP_DB_DIC).

URL :

http://.../ServAut.woa/wa/DADBDico

Paramètres :

(Cette action ne prend aucun paramètre.)

 Action : Cryptage de mot de passe

Description :

Cette action permet d'accéder au formulaire de saisi et de cryptage des mots de passe. La valeur obtenue peut être utilisée pour définir le mot de passe d'accès à l'application. Voir le paramètre APP_ADMIN_PASSWORD dans la configuration de l'application.

URL :

http://.../ServAut.woa/wa/DAPasswd

Paramètres :

(Cette action ne prend aucun paramètre.)

 Action : XML pour ZAP

Description :

Cette action permet de générer et récupérer un document XML donnant la description des applications. La description obtenue peut être utilisée pour initialiser l'application ZAP.

Le contenu de XML est définit via l'interface fournie par l'application ServAut (menu ZAP). L'URL d'accès à cette action peut être utilisée pour initialiser le contenu de ZAP dynamiquement.

URL :

http://.../ServAut.woa/wa/getXMLZAP

Paramètres :

(Cette action ne prend aucun paramètre.)

 Services Web

L'application définit deux actions "db" et "usr" utilisées pour récupérer les dictionnaires de connexion et les informations sur les utilisateurs. Ces actions utilisent le protocole SOAP et elles échangent les informations cryptées.

Ces actions ne doivent pas être accédées directement. Les méthodes de la classe SAUTCLient (framework CRIWebApp) doivent être utilisées pour communiquer avec ces actions.
